== Lecons apprises

* axe-core via Playwright headless detecte reellement les problemes de contraste WCAG 2.1 AA
* Bootstrap 5.3 utilise ses propres variables `--bs-*` qui ne suivent PAS les custom CSS vars — il faut les overrides explicites
* Les `<p>` vides generes par JBake AsciiDoc ont `rect.height=0` et declenchent des faux positifs dans les tests responsive — les filtrer
* Les CDN (Font Awesome) ne sont pas disponibles en mode `file://` pour les tests Playwright — preferer Bootstrap Icons deja charge
* Les couleurs d'accent claires (#6ea8fe, #0d6efd) sur fond sombre avec texte blanc echouent WCAG AA 4.5:1 — utiliser des couleurs plus sombres pour les fonds d'admonition
* Le pattern cheroliv pour le navbar : `body { padding-top: var(--navbar-height) }` + `html { scroll-padding-top: var(--navbar-height) }` + `.main-content-padding` est robuste
* `<main>` landmark est requis par WCAG 2.1 — JBake/Thymeleaf ne le genere pas par defaut
