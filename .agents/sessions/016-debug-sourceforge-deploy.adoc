== Lecons apprises

* Les secrets GitHub Actions multi-lignes (cles SSH) perdent leurs newlines avec `echo` — toujours encoder en base64 et decoder dans le workflow
* `echo "$VAR" > file` ne preserve pas les newlines dans GitHub Actions (shell /usr/bin/bash -e)
* `echo "$VAR" | base64 -d > file` est le pattern standard pour les cles SSH en CI/CD
* Un tag GitHub est le declencheur naturel du pipeline CI/CD, pas juste pour les releases
